Base URL: /api/marketing-person/{user_code}
Auth: Authorization: Bearer <JWT> (middleware multi_jwt:api)
GET /personal/expenses

Purpose: Return paginated personal expenses for the marketing person.
URL: /api/marketing-person/{user_code}/personal/expenses
Query params:
perPage: integer (default 25)
page: integer
section: string
month: 1–12
year: YYYY
search: string (searches description and section)
Success (200): JSON with data.items[] and data.meta (pagination).
Example curl:
curl -H "Authorization: Bearer <TOKEN>" "https://your-domain.test/api/marketing-person/MKT001/personal/expenses?perPage=10"
POST /personal/expenses

Purpose: Create a personal expense (supports file upload).
URL: /api/marketing-person/{user_code}/personal/expenses
Content-Type: multipart/form-data
Form fields:
section: string (optional)
expense_date: date (YYYY-MM-DD, optional — defaults to today)
amount: numeric (required)
description: string (optional)
file: file (optional — jpg,jpeg,png,pdf, max 10 MB)
Success (201): JSON with created expense object (id, section, expense_date, amount, description, file_url, status).
Example curl (upload):
curl -X POST -H "Authorization: Bearer <TOKEN>" -F "amount=1200" -F "expense_date=2025-12-13" -F "section=Travel" -F "description=Parking fee" -F "file=@/path/to/receipt.jpg" "https://your-domain.test/api/marketing-person/MKT001/personal/expenses"
Validation & Errors

Validation: amount required; file max 10240KB and mime jpg,jpeg,png,pdf. Validation errors return HTTP 422 with Laravel validation payload.
Status field: 0 = pending, 1 = approved, 2 = rejected.
